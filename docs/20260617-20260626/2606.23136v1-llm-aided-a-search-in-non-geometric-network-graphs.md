## 摘要
在非几何网络图中寻找最短路径，其中边权重编码的是延迟或货币成本等任意度量而非空间距离，这对启发式搜索算法构成了挑战。这些算法的效率依赖于信息丰富的启发式函数，在空间域中通常由几何距离提供，但在非几何图上则没有对应的几何距离。我们提出了一种大语言模型（LLM）辅助的A*算法，其中LLM生成中间路径点，引导A*扩展至有希望的图区域。该方法的核心是地标距离，它既作为可采纳的基于地标的（ALT）启发式函数，又作为一种紧凑的结构特征提供给LLM，恢复了其在非几何图上原本缺乏的到达目标的距离信号。我们在多种拓扑结构、多达2000个节点的图上进行了全面实验，结果表明，LLM生成的路径点将扩展节点数减少了约50%，同时相比最优解仅带来微小的路径成本增加。我们进一步分析了提示工程的影响，并表明引入紧凑的结构特征，即启发式估计值，比高级提示技术更有效。这些发现展示了将LLM指导与经典搜索算法相结合用于高效网络优化的潜力。

## Abstract
Finding the shortest path in non-geometric network graphs, where edge weights encode arbitrary metrics such as latency or monetary cost rather than spatial distance, poses a challenge for informed search algorithms. Their efficiency depends on an informative heuristic, typically supplied in spatial domains by geometric distances that have no counterpart on non-geometric graphs. We propose a large language model (LLM)-aided A* algorithm in which an LLM generates intermediate waypoints that guide the A* expansion toward promising graph regions. At the core of the approach are landmark distances, which serve both as an admissible landmark-based (ALT) heuristic for the search and as a compact structural feature that, supplied to the LLM, restores the distance-to-destination signal it would otherwise lack on non-geometric graphs. Our comprehensive experiments on multiple graph topologies with up to 2,000 nodes demonstrate that LLM-generated waypoints reduce the number of expanded nodes by around 50% while incurring only a marginal path cost increase compared to the optimal solution. We further analyze the impact of prompt engineering and show that incorporating compact structural features, namely heuristic estimates, is more effective than advanced prompting techniques. These findings demonstrate the potential of combining LLM- based guidance with classical search algorithms for efficient network optimization.

## 2. 论文提出的方法论

- **核心思想**：用 LLM 为给定的源‑目标对生成一系列中间路径点（waypoints），通过将这些路径点作为子目标注入 A* 的评估函数，引导搜索向有希望的区域扩展，减少不必要的探索。
- **地标距离的双重角色**：
  - **作为可采纳启发式（ALT）**：利用三角形不等式，通过预计算一小部分地标节点的最短路径距离，构造可采纳的下界估计 \(h(n, d)\)，用于 A* 搜索本身。
  - **作为 LLM 的结构特征**：将每个节点到目标的目的地启发式估计值 \(h(n, d)\) 提供给 LLM，作为紧凑的距离信号，弥补 LLM 在非几何图上无法直接感知“到达目标有多远”的不足。
- **算法流程**（Algorithm 1）：
  - 输入：图 \(G\)，源 \(s\)，目标 \(d\)，启发式函数 \(h\)。
  - LLM 根据提示（含邻接表、启发式值等）生成有序路径点序列 \(\psi_1, \psi_2, \dots, \psi_k\)。
  - 初始化优先级队列，起始活动的路径点为 \(\psi_1\)。

  - 搜索持续到目标被取出或队列为空。
- **提示设计**：提供了多种提示配置，包括仅邻接表、邻接表 + 启发式值、以及结合链式思维（CoT）或少样本（few‑shot）的高级提示技术，以分析提示信息对路径点质量的影响。
- **可采纳性讨论**：标准 A* 的可采纳性由 ALT 启发式保证；加入路径点项后整体搜索不再严格可采纳，但论文通过实验展示路径成本增加极小，并计划在未来工作中给出次优性界限。

## 6. 论文的主要结论与发现

- **效率大幅提升**：LLM 生成的路径点使扩展节点数平均减少约 **50%**，大幅优于标准 A*，且拓展了算法在大图上的可扩展性。
- **路径质量牺牲极小**：相比最优路径，路径成本增加仅为 **0.34–0.66**（无权图下的跳数），远低于贪婪最佳优先搜索的 **3.92**。这表明 LLM 仅作为导引，不牺牲最终解的质量。
- **提示信息优于提示技术**：在提示中提供紧凑的结构特征（启发式估计值）比使用 CoT 或少样本等高级提示策略更为有效。后者仅在缺乏启发式值时能大幅提升效果，但一旦提供了结构信息，其边际收益趋于饱和。
- **路径点数量需适度**：过多的路径点会限制搜索的灵活性，降低收益。数量应与图直径和路径长度相匹配。
- **鲁棒性分析**：即使 LLM 生成不合理的路径点，A* 仍会依据其代价函数和启发式搜索到近优解，错误路径点主要影响探索开销而非最终路径质量。

## 7. 优点

- **创新的融合方式**：首次将 LLM 作为导引与经典 A* 搜索结合用于非几何图的最短路径，成功将 LLM 的全局推理能力与 A* 的完备性优势互补。
- **地标距离的双重利用**：巧妙地让同一套地标距离既作为搜索的可采纳启发式，又作为给 LLM 的紧凑距离信号，一举解决了非几何图上启发式缺失和 LLM 缺乏空间感知的两大难题。
- **扎实的提示工程分析**：通过系统的消融实验，揭示了“提示内容”（结构特征）比“提示技巧”（CoT、少样本）更重要的洞见，为后续研究指明了方向。
- **实用的鲁棒性**：LLM 仅作为优先级引导，路径解由 A* 保证，因此 LLM 的生成偏差不会导致致命错误，适用于要求可靠性的网络场景。

## 8. 不足与局限

- **仅局限于无权图**：当前所有实验均在边成本为统一跳数的无权图上进行，未涉及加权、资源受限的最短路径问题（如服务功能链路由），限制了直接应用到实际网络权重场景的能力。
- **可采纳性未严格保证**：加入路径点项后搜索不再具有可采纳性，虽然实验显示路径成本增加极小，但理论上的次优界尚未给出。
- **缺少端到端时间对比**：论文仅报告扩展节点数，未测量 LLM 推理带来的额外时间开销，无法完整评估实际部署时的效率。
- **LLM 依赖与成本**：方法需要调用商业 LLM API，带来延迟、费用和隐私问题；对于大规模在线路由场景，频繁调用 LLM 可能不切实际。

- **路径点生成的敏感性与可解释性**：LLM 输出可能对提示细节敏感，路径点选择背后的推理不透明，缺乏可解释性，可能影响运维人员的信任。
