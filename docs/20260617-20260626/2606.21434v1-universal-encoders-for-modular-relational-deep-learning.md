## 摘要
关系深度学习（RDL）将多表格数据库建模为时态异构图，以实现端到端的表示学习。尽管RDL发展迅速，现有方法仍面临显著的泛化障碍：它们要么是特定于模式的，需要为每个新数据库从头训练，要么依赖于将特征编码与图消息传递纠缠在一起的单体架构。通过分析这些局限性，我们确立了构建基础关系模型的四大支柱：语义粒度、结构拓扑、时序因果和统一优化。针对这些支柱，我们提出一种模块化方法，将行编码与图消息传递解耦。我们引入了通用行编码器，这是一个基于Transformer的模块，它将原始单元格数据与模式元数据（包括列语义、表名和全局分布统计）相融合，以产生表宽不变的行嵌入。通过明确地将全局统计信息馈送至行内自注意力机制，该编码器能够原生地上下文化未见特征并处理稀疏数据。作为任何下游图架构的灵活“后端”，我们的预训练编码器在已建立的RelBench基准上增强了跨数据库知识迁移，同时改善了学习收敛性和内存占用。

## Abstract
Relational Deep Learning (RDL) models multi-tabular databases as temporal heterogeneous graphs for end-to-end representation learning. While RDL is evolving rapidly, existing approaches face significant generalization obstacles. They are either schema-specific, requiring training from scratch for every new database, or they rely on monolithic architectures that entangle feature encoding with graph message-passing. Analyzing these limitations, we establish four core pillars for building foundational relational models: semantic granularity, structural topology, temporal causality, and unified optimization.   Addressing these pillars, we propose a modular approach that decouples row encoding from graph message-passing. We introduce the Universal Row Encoder, a transformer-based module that integrates raw cell data with schema metadata$-$including column semantics, table names, and global distribution statistics$-$to produce table-width invariant row embeddings. By explicitly feeding global statistics to an intra-row self-attention mechanism, the encoder natively contextualizes unseen features and handles sparse data. Serving as a flexible "backend" for any downstream graph architecture, our pretrained encoder enhances cross-database knowledge transfer on the established RelBench benchmarks while improving learning convergence and memory footprint.

## 二、论文提出的方法论
### 2.1 概念框架：四大支柱
为构建可跨数据库迁移的关系基础模型，论文提出四项核心复杂性（称为“支柱”）：
1. **语义与特征粒度**：跨表特征分布差异、表内特征语义关系、稀疏数据处理。
2. **结构与关系拓扑**：主外键语义区分、枢纽节点导致的过平滑、严格的模式无关性。
3. **时序因果**：连续时间表示、严格因果约束、多表异步动态、概念漂移。
4. **统一优化与可迁移性**：多任务统一目标、采样子图表示标准化、损失/梯度尺度平衡。

### 2.2 模块化架构（核心技术贡献）
将预测流程分解为三个独立组件：
```
hv = fθ(rv)          # 通用行编码器，输出固定维度 d 的行嵌入
{zu} = gφ(GS, {hv})  # 图模块，在时序采样子图上进行消息传递
ŷu = dψ(zu)          # 任务特定预测头
```
- **fθ**：完全与数据库模式无关的行嵌入器。
- **gφ**：可使用同构或异构图神经网络，严格保证时序因果采样。
- **dψ**：支持多任务学习，通过损失加权 λℓ 平衡梯度。

### 2.3 通用行编码器（Universal Row Encoder）
编码器将任意数据库行转化为固定维度嵌入，其输入包含：
```
r = (vj, cj, t, sj, δj) for j=1..m
```
- vj：原始单元格值
- cj：列名
- t：表名
- sj：预计算的列统计量（数值列：均值、标准差、最大/最小、四分位数等；分类列：基数与预计算文本嵌入；时间列：最早/最晚时间等），**在时间上按训练/过去数据分割计算**以防止信息泄露。
- δj：缺失指示符

**处理流程**：
1. **多信息融合**：对每列构造上下文嵌入  
   xj = fstype(j)(vj) + gname(cj) + gtable(t) + gstats(sj) + gmiss(δj)  
   分别用语义类型编码器、文本嵌入（GloVe）、线性投影等方式实现。
2. **行内自注意力**：将 (x1,…,xm) 输入一个**无位置编码的Transformer**，使其对列序置换等变，生成上下文列表示 (z1,…,zm)。
3. **池化与投影**：通过 sum pooling、max pooling、注意力池化拼接后线性映射，得到固定大小行嵌入 h ∈ R^d。  
该设计避免了对整库 N² 的全局注意力，计算复杂度仅与列数 m 有关。

### 2.4 图模块与时序因果
- 图中节点 v 的初始表示均为 h_v（统一特征空间）。
- 消息传递仅在满足 τ(v) ≤ τ(u) 的时序子图上进行，不泄露未来信息。
- 支持同构 GNN（参数共享）与异构 GNN（关系特定参数），保持架构灵活性。

### 2.5 预训练策略
- **仅编码器预训练**：冻结通用行编码器，作为特征提取器用于新数据库。
- **端到端预训练**：编码器、GNN、任务头联合预训练，让行编码器接收来自下游结构与时序模式的梯度。

### 3.2 对比方法

- **通用编码器变体**：

- 预训练：最大 50k 步，线性预热（前 10%）+ 余弦退火。

- 模型超参数：Transformer 1/2/4 层，8 注意力头，dropout 0.1；GraphSAGE 2 层，隐藏维度 512；预测头仅一层全连接。

## 六、论文的主要结论与发现
1. **模块化解耦有效**：将行编码与图传播分离，可显著提升跨数据库迁移能力，预训练的通用行编码器能作为“即插即用”后端。
2. **预训练带来快速适应**：冻结编码器加下游 GNN 训练，在多数分类任务上性能接近甚至超过从头训练 5000 步的任务特定模型，且仅需少量步骤。
3. **微调收益有限**：冻结编码器与微调编码器在多数任务上差异微小，表明预训练已捕获大部分可迁移语义特征。
4. **显著内存效率**：通用模型比专用模型内存缩小最高约 10 倍（如 rel-trial 从 368.81 MB 降至 34.06 MB），预测精度仅有温和下降。
5. **同构 GNN 后端稳定**：共享消息传递参数的同构 GNN 在预训练中损失收敛更稳定，精度与异构 GNN 接近。

## 七、优点
- **清晰的框架**：四大支柱为关系基础模型提供了结构化的设计维度，有助于后续研究聚焦。

- **工程实用性强**：预训练编码器可冻结使用，内存效率极高，适合在多个数据库上同时部署。
- **实验验证扎实**：采用严格留一数据库评估，从精度、收敛、内存多角度论证，开源代码增强了可复现性。
- **统一优化思路**：模块化设计天然适配多任务学习与损失平衡，为未来扩展奠定基础。

## 八、不足与局限
- **主要解决第一支柱**：当前实现主要针对语义粒度，对结构拓扑、时序因果、多任务统一解码的处理相对简单（仅依靠显式图构建和时序采样），未在模型架构中内化。
