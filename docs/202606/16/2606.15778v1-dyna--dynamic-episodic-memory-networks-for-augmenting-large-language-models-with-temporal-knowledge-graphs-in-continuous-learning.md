## 摘要
大型语言模型（LLM）在纳入新知识时难以避免遗忘或高昂的重新训练。我们提出DYNA，一个轻量级框架，利用时态知识图谱增强冻结的LLM，其中事件为节点，时态关系为有向、带时间戳的边。该图谱作为外部可更新记忆。查询时，DYNA通过随机游走和中心性度量检索相关节点，进而增强LLM的响应。在三个时态回忆任务上的评估显示，与微调相比，DYNA将灾难性遗忘减少约7%，与标准RAG相比，时间排序改善约5%。较高的图聚类系数与更好的检索效果相关，表明图结构的重要性。主要贡献：（1）将情景记忆构建为时态知识图谱；（2）无需重新训练的LLM增强方法；（3）将图属性作为检索性能的预测因子。

## Abstract
Large Language Models (LLMs) struggle to incorporate new knowledge without forgetting or costly retraining. We propose DYNA, a lightweight framework that augments a frozen LLM with a temporal knowledge graph where events are nodes and temporal relations are directed, timestamped edges. The graph serves as an external, updatable memory. At query time, DYNA retrieves relevant nodes via random walks and centrality measures, then augments the LLM's response. Evaluated on three temporal recall tasks, DYNA reduces catastrophic forgetting by ~7% compared to fine-tuning and improves temporal ordering by ~5% over standard RAG. Higher graph clustering coefficients correlate with better retrieval, showing that graph structure matters. Contributions: (1) episodic memory as temporal KG, (2) retraining-free LLM augmentation, (3) graph properties as predictors of retrieval performance.

### 2. 论文提出的方法论
- **核心思想（DYNA框架）**：
  - 将LLM视为一个**冻结的**（参数不变）核心推理引擎。
  - 构建一个**时态知识图谱**作为外部情景记忆，该图谱随新事件的到来持续增长。
  - 查询时，通过图谱的探索和检索机制，找到与查询相关的结构化上下文，并将其与LLM的内部知识结合生成最终回答。
- **关键技术细节**：
  - **图谱构建**：
    - **节点**：每一个新事件被编码为一个节点，存储事件的文本描述。
    - **有向边**：基于事件时间戳，定义三种时序关系：
      - `before` (vᵢ → vⱼ):  事件i早于事件j。
      - `after`：反向推导，一般不予显式存储。
      - `co-occurs`：发生在短时间窗口内事件的共现关系。
    - **边权重**：采用指数衰减函数 `w(vᵢ, vⱼ) = exp(-λ |tᵢ - tⱼ|)`，赋予时间上更邻近的事件更高的权重，较久远的事件关联性则指数级下降。
  - **图谱维护（剪枝策略）**：
    - **基于时间的剪枝**：移除超过时间阈值 `T_max` 的旧节点。
    - **基于度的剪枝**：移除连接数少的低重要性节点。
  - **检索机制**：
    - **种子节点选择**：通过词法或嵌入相似度，为查询找到图谱中最相关的`top-k`节点作为起点。
    - **随机游走遍历**：从种子节点出发，沿经时间权重加权的边进行多次随机游走，以捕捉不仅直接相关、而且在时间上邻近的事件。
    - **中心性打分**：访问到的节点将综合以下指标进行评分，以确定其最终相关性：
      - 度中心性。
      - 接近中心性。
      - 在随机游走中的被访问频率。
      - 最终得分融合公式：`score(v) = α · degree(v) + β · closeness(v) + γ · visit_freq(v)`。
  - **查询增强与生成**：按照“系统指令 + 检索到的带时间戳事件 + 用户原始查询”的范式构建增强提示，交由冻结的LLM生成回答。

### 8. 不足与局限
- **检索瓶颈**：检索精度（尤其是首位精度`P@1`仅为21.28%）是当前最大的性能瓶颈，意味着检索机制经常找不准最相关的事件，限制了LLM推理能力的发挥。
- **推理能力限制**：在需要**隐式推理**的困难任务（如TimeQA Hard）上，其优势明显缩小。这表明时态图谱对已明确结构化关系的任务更有帮助。
- **冷启动问题**：在**图谱规模较小**的初期阶段（如前300个事件），DYNA的性能低于某些依赖预训练嵌入的基线方法。
- **应用场景有限**：

- **评估范围**：模型测试仅基于一个LLM（LLaMA 3），所用数据集相比充满噪声的真实世界数据流仍较为理想化，泛化性有待验证。
