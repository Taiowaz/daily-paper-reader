## 摘要
小语言模型为大规模系统提供了高效的替代方案，但其在多个依赖决策下执行结构化算法的能力仍未被充分理解。我们将图算法执行研究为一个闭环预测问题，其中模型根据当前图和算法状态反复选择下一步动作。我们的评估框架涵盖了多种经典图过程、多个人工图族以及互不重叠的训练、验证和测试分区。评估既包含局部决策质量，也通过步骤准确率、精确 rollout 准确率、约束有效性、部分解质量、前缀生存分析和基于干预的诊断来考察全局执行行为。结果表明，适应过程可以为遍历和着色等结构类过程生成可靠策略，而加权算法对误差累积的敏感性依然显著更高。更广泛而言，这些发现表明，强大的下一步预测并不必然转化为可靠的自主执行，因此评估算法型语言模型应通过完整的闭环 rollout 而非孤立决策来进行。

## Abstract
Small language models offer an efficient alternative to large-scale systems, but their ability to execute structured algorithms over multiple dependent decisions remains poorly understood. We study graph algorithm execution as a closed-loop prediction problem in which a model repeatedly selects the next action from the current graph and algorithmic state. Our evaluation framework covers several classical graph procedures, multiple synthetic graph families, and disjoint training, validation, and test partitions. It assesses both local decision quality and global execution behaviour using step accuracy, exact rollout accuracy, constraint validity, partial solution quality, prefix survival, and intervention-based diagnostics. The results show that adaptation can produce reliable policies for structural procedures such as traversal and coloring, while weighted algorithms remain substantially more sensitive to error accumulation. More broadly, the findings demonstrate that strong next-step prediction does not necessarily translate into reliable autonomous execution and motivate evaluating algorithmic language models through complete closed-loop rollouts rather than isolated decisions.

# 小语言模型闭环图算法执行：步骤准确性与 rollout 可靠性
## 论文核心问题与整体含义
- 小语言模型（SLMs）在多步依赖决策下执行经典图算法的可靠性尚未被系统研究。
- 传统评估仅看孤立步骤准确率（teacher-forced），但实际执行中模型自身的错误会改变后续状态，导致严重累积失效。
- 本文将图算法执行建模为**闭环预测问题**：模型根据当前图和算法状态预测一个动作，由外部确定性执行器更新状态，形成闭环反馈。
- 核心主张：**强下一步预测能力不一定转化为可靠自主执行**，评估算法型语言模型应基于完整闭环 rollout，而非孤立决策。

## 方法论
- **闭环公式化**：
  - 图 \(G=(V,E,w)\)，算法状态 \(s_t\)，动作 \(a_t\)，转移函数 \(s_{t+1}=T_A(G,s_t,a_t)\) 由外部执行器实现。
  - 语言模型作为策略 \(\pi_\theta(a_t | x(G,s_t))\)，输出一个可解析的紧凑动作行。
  - 评估双轨制：
    - **教师强制（teacher-forced）步骤评估**：仅在参考正确状态 \(s_t^*\) 上查询模型。
    - **自主 rollout 评估**：从初始状态开始，每一步使用模型自身预测的动作推进，直至终止或失败。
- **训练**：
  - 使用参数高效微调（LoRA），基础模型为 Llama-3.2-1B-Instruct 和 Qwen2.5-1.5B-Instruct，均量化至 4-bit NF4。
  - 每个模型-算法组合独立训练一个适配器，仅对动作 token 计算因果语言模型损失。
  - 训练细节：rank 16，缩放因子 32，dropout 0.05；AdamW，学习率 \(2\times10^{-4}\)，余弦调度，3% 预热，梯度裁剪 1.0，有效批次大小 16，最多 8 epoch，早停。
- **动作空间**（表 1）：紧凑且可自动生成监督信号。
  - BFS/DFS：入队/压入新发现的邻居。
  - 着色：为当前顶点分配颜色。
  - Prim/Borůvka：选择一条割边/组件边。
  - Dijkstra：选择访问顶点并松弛距离。
- **基线**：评估未适配的指令模型（零样本和思维链提示），均无任何一次精确 rollout 成功。

## 实验设计
- **数据集**：TinyGraphEstimator 集合中的 1200 个无向图，20–30 个顶点，平衡分配至 Barabási–Albert、Erdős–Rényi 和 Watts–Strogatz 三类生成器。训练 900 图、验证 150 图、测试 150 图，拆分时确保无生成来源泄露。
- **加权算法**：Prim、Borůvka、Dijkstra 使用确定性整数权重（1–10）。
- **评估指标**：
  - **步骤准确率**：参考状态下的精确动作匹配率。
  - **精确 rollout 准确率**：完整闭环执行后最终答案与参考解完全一致。
  - **约束有效性**：任务特定的弱结构性检查（如无冲突着色、可达顶点序列、形成生成树等），**不**等价于最优解。
  - **软分数**：部分一致度（顺序位置准确率、着色准确率、边 F1、距离准确率等）。
  - **rollout 诊断**：步-轨迹差距、前缀生存 AUC、首次错误位置、干预纠正次数（模拟以 oracle 动作校正后所需的替换量）。
- **对比方案**：同一架构的未适配模型（零样本、CoT），以及与适配后模型的全面对照。

## 主要结论与发现
- 遍历类算法（BFS、DFS）和着色在对 SLM 进行适配后，可实现 **>94% 的精确 rollout 准确率**，且步骤准确率与 rollout 准确性高度一致。
- 加权算法（Borůvka、Dijkstra、Prim）则呈现**严重的步-轨迹差距**：步骤准确率可达 83–93%，但精确 rollout 准确率骤降至 13–41%。
- 诊断指标揭示：
  - Dijkstra 在极早期即发生错误，前缀生存 AUC 仅 15–18%，平均需 3–4 次 oracle 干预才能修正。
  - Borůvka 前缀 AUC 约 70%，表明多数运行初期正确但无法维持至终止。
  - 局部错误通过改变组件结构或距离映射，对后续决策造成全局性破坏。
- 零样本和 CoT 提示在未适配模型中**无法产生任何一条完整正确轨迹**，显示仅靠指令遵循远不足以完成闭环执行。
- 总体结论：**预测质量的聚合指标（如步骤准确率）会严重高估实际运行的可靠性**；闭环 rollout 应作为评估算法型语言模型的主要单元。

## 优点
- **闭环架构清晰**：将推理过程分离为“模型策略”与“确定性执行器”，精准暴露累积性误差。
- **评估体系完整**：同时使用教师强制和闭环衡量，引入前缀生存和干预诊断，超越常规的准确率对比。
- **任务与模型多样**：覆盖六种经典图算法和两种 SLM 架构，且包含结构性与加权算法的对比。
- **无适配基线严谨**：确认了当前泛用指令模型的零样本闭环能力几乎为零，增强了结论的说服力。

## 不足与局限
- **图域受限**：所有图均为 20–30 顶点的合成图，且测试仅在同一生成器族内进行，未验证对更大规模图、真实世界图的泛化。
- **权重与参考解单一**：权重为确定性整数，且精确 rollout 准确率仅针对一种参考实现及其破平则约定，排除了其他合法等优解。
- **约束有效性指标较弱且异质**：仅检查可行结构而非最优性，无法替代实际任务正确性。
- **训练细节不完整**：缺乏多训练种子、超参数敏感性分析及跨算法共享适配器的探索。
- **算力报告缺失**：未说明 GPU 型号与训练耗时，不利于资源估算与复现。
- **实际应用局限**：研究限于图算法的执行闭环，尚未涉及工具使用、规划等多步智能体场景。
