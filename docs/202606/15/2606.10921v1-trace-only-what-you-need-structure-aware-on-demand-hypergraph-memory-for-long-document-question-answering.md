## 摘要
长文档问答要求大语言模型对散布在冗长文档中的证据进行推理，答案往往依赖于事件顺序、章节级上下文以及跨部分证据关联。尽管检索增强生成通过检索相关证据减少了输入上下文，但现有的结构化检索增强生成方法仍面临三个局限：昂贵的查询无关知识组织、原文档结构利用不足以及历史推理经验无法复用。为应对这些局限，我们提出DocTrace，一个支持查询触发式知识组织、文档结构感知和经验引导推理的多智能体检索增强生成长文档问答框架。DocTrace通过轻量级文档结构树索引保留文档层级，在推理过程中按需构建智能体共享的超图结构化工作记忆，并将成功的推理方案存储在图结构经验记忆中以便将来复用，从而实现对相关长文档问题的自适应探索。在四个长文档问答数据集上的实验表明，DocTrace在三个数据集上取得最佳性能，超越最强基线ComoRAG，F1最高提升8.85%，EM最高提升4.40%，同时总计算成本降低53.32%

## Abstract
Long-document question answering (QA) requires large language models (LLMs) to reason over evidence scattered across lengthy documents, where answers often depend on event order, section-level context, and cross-part evidence connections. Although retrieval-augmented generation (RAG) reduces the input context by retrieving relevant evidence, existing structured RAG methods still face three limitations: costly query-agnostic knowledge organization, insufficient use of original document structure, and no reuse of historical reasoning experience. To address these limitations, we propose DocTrace, a multi-agent RAG framework for long-document QA that supports query-triggered knowledge organization, document-structure-aware and experience-guided reasoning. DocTrace preserves document hierarchy with a lightweight document structural tree index, constructs agent-shared hypergraph-structured working memory on demand during reasoning, and stores successful reasoning plans in graph-structured experience memory for future reuse, enabling adaptive exploration across related long-document questions. Experiments on four long-document QA datasets show that DocTrace achieves the best performance on three datasets, surpassing the strongest baseline, ComoRAG, by up to 8.85% in F1 and 4.40% in EM, while reducing the overall computational cost by 53.32%

## 1. 核心问题与研究背景
- **长文档问答的挑战**：长文档（如书籍、报告、叙事文本）中的答案往往依赖于事件顺序、章节级上下文和跨段落证据关联，直接将整个文档输入大语言模型（LLM）会导致性能下降。
- **现有结构化RAG的不足**：
  - **查询无关知识组织成本高**：多数方法在回答任何问题之前就预先构建文档级摘要树或知识图谱，带来巨大延迟和 token 开销。
  - **文档结构利用不足**：常见做法是将文档切成扁平文本块，破坏了章节边界、段落顺序等原始结构信息。
  - **历史推理经验无法复用**：复杂多步推理的轨迹在回答完一个问题后即被丢弃，无法为后续相似问题提供参考。
- **整体含义**：论文提出 DocTrace 框架，旨在以查询触发、结构感知、经验引导的方式解决上述三大局限。

## 2. 方法论
DocTrace 是一个多智能体 RAG 框架，核心思想是**按需构建工作记忆**，**保留文档结构**，并**复用成功推理方案**。

### 2.1 整体架构
- **Orchestrator-Investigator 多智能体**：Orchestrator 负责问题分解、计划制定与最终答案合成；Investigator 负责单个子问题的证据检索与子答案生成。
- 三大部分：**文档结构树索引 (T)**、**超图工作记忆 (M)**、**图结构经验记忆 (DM)**。

### 2.2 关键技术组件
- **文档结构树索引 (T)**：
  - 通过 LLM 识别文档标题模式，将文档解析为 `document → chapter → section → paragraph` 的四级树结构。
  - 每个节点存储文本内容及子节点预览，并利用 spaCy 进行命名实体识别，建立节点-实体链接。
  - 支持 **TreePPR** 算法：在约束范围（scope）和检索形状（shape，如紧凑、顺序、分散）下，以 Personalized PageRank 传播种子节点相关性，实现结构感知的节点排序。

- **查询触发的超图工作记忆 (M)**：
  - 不预先构造全文档知识结构，而是在推理过程中由 Investigator 按需构造，代理间共享且跨子问题持久化。
  - **抽象视图 (MA)**：对选定的树节点进行自底向上层级摘要，在控制每组摘要规模的前提下保留文档位置信息；支持摘要复用与 drill-down 至原始段落。
  - **关系视图 (MR)**：仅对当前子问题相关的树节点提取 n 元关系与实体，构建局部超图；支持关系推理路径检索，并与已有关系合并。

- **图结构经验记忆 (DM)**：
  - 存储成功推理的**有向无环图（DAG）**（分解的子问题及依赖关系）。
  - 每个 DAG 与问题骨架、推理签名关联，支持多路径检索（骨架相似、签名相似、结构重叠）。
  - 允许将子问题的 DAG 动态拼接到父问题中，生成新的组合示例，实现经验迁移。

### 2.3 流程概要
1. **Orchestrator** 对问题抽象化（提取主题实体、骨架、推理签名），选取种子节点，从 DM 检索相关 DAG 示例，初始化多个推理计划。
2. 每个计划按拓扑排序分配给 **Investigator**，Investigator 分析检索几何（范围、形状、粒度），通过 TreePPR 定位节点集，根据粒度选择工作记忆视图进行回答：
   - 关系粒度 → 构建/复用 MR，进行关系路径检索。
   - 其他粒度 → 若节点集合适，直接阅读回答；若超出预算，则通过 MA 摘要回答并支持 drill-down。
3. Orchestrator 根据子答案更新 DAG，最终从完成的所有 DAG 中选出最优答案，并将高质量 DAG 存入 DM。

## 6. 主要结论与发现
- DocTrace 在**三个数据集上取得最佳**性能，尤其在 NarrativeQA 上 F1 超过 ComoRAG 8.85%，EM 超过 4.40%；在 EN.QA 的 F1 上也领先，但在 EN.MC 的 ACC 上略逊于 ComoRAG（差 2.63%），其中去掉 DM 的版本在 DetectiveQA 上最优。

- 文档结构树索引和按需超图工作记忆对性能至关重要，去除其中任一部件均导致明显下降。
- 经验记忆的在线学习能够稳定提升多轮问答的累积准确率，并支持跨数据集的推理模式迁移（虽有一定领域错配风险）。
- Token 成本不受文档长度显著影响，具有很好的规模可扩展性。

## 7. 优点与方法亮点
- **查询触发式知识组织**：避免预建索引的高额开销，使计算资源集中于问题相关区域。
- **原生文档结构利用**：用四级结构树保留章节/段落层级，检索与摘要均在结构引导下进行，有利于恢复叙事流。
- **经验记忆复用**：将推理 DAG 以图结构存储，支持组合和泛化，提升相似问题的推理效率与质量。
- **多粒度工作记忆**：抽象视图与关系视图互补，灵活应对不同证据需求。
- **高效且可扩展**：在提升性能的同时大幅降低整体 token 消耗，且对长文档的边际成本稳定。

## 8. 不足与局限
- **多模态缺失**：仅支持纯文本，未考虑表格、图像等多模态元素。
- **对 LLM 依赖性**：规划、问题抽象、摘要与答案生成仍需多次调用大模型，虽然较基线低，但仍有较高 API 成本。
- **多选任务表现不突出**：在部分多选数据集（如 EN.MC）上略低于最强基线，可能需要额外的推理桥接能力。
- **跨域迁移有限**：经验记忆的跨数据集泛化效果存在波动，不同域间推理模式不完全适配。
- **在线学习初始化**：经验记忆启动初期性能较低且波动，需要积累一定数量的成功经历才能发挥作用。
- **未涉及训练**：整体框架均基于提示与检索，不包含参数微调，可能限制在某些场景下的进一步优化空间。
