## 摘要
我们提出GRAB，一种用于表问答的构造-编码-桥接流水线。该方法将关系数据提升为异构图，通过消息传递进行编码，并借助一组查询条件化的隐式令牌将信号传递给大语言模型。这为大语言模型提供了紧凑的任务相关结构表示以及展平文本。关键之处在于，大语言模型严格保持冻结以保留其通用推理能力；我们仅训练轻量级图编码器和隐式桥接（91M参数），使得整个流水线能够高效训练。我们的流水线显著提升了关系问答的性能，在需要多表的挑战性场景中增益最大，为连接关系深度学习与大语言模型提供了一条高效且原则化的途径。

## Abstract
We introduce GRAB, a constructor-encoder-bridge pipeline for table question answering. Our method lifts relational data into an heterogeneous graph, encodes it via message passing, and transfers the signals to an LLM through a small set of query-conditioned latent tokens. This provides the LLM with a compact, task-relevant structural representation together with the flattened text. Crucially, the LLM remains strictly frozen to preserve its general reasoning capabilities; we train only the lightweight graph encoder and latent bridge (91M parameters), allowing the entire pipeline to be trained efficiently. Our pipeline significantly improves performance on relational Question Answering, with the largest gains in demanding multi-table settings, offering an efficient, principled way to connect relational deep learning with LLMs.

## 论文核心问题与研究意义
*   **核心问题**：多表问答任务中，大语言模型通常将表格序列化为纯文本输入，丢失了表格的行列结构、值重复、外键连接等关系信息，导致在结构复杂、跨表推理时表现脆弱。
*   **整体目标**：设计一种既能保留表格关系结构，又不破坏 LLM 通用推理能力的高效多模态接口，使 LLM 能借助轻量外部模块获得紧凑的、任务相关的结构表示。

## 方法论与关键技术
**总体框架（GRAB）**：构造器→编码器→桥接，注入冻结 LLM。
*   **图构造器（Graph Constructor）**：
    *   将关系数据库 \(T\) 和外键元数据 \(F\) 映射为异构图。
    *   节点类型：行节点、列类节点（外键关联的列合并为同一类）、值组节点（分类/文本值去重，数值按分位数桶化）。
    *   边：行–值边（包含关系）、列–值边（列包含哪些值）。
    *   通过共享值节点和外键合并列类，显式表达重复值和连接关系。
*   **图编码器（Graph Encoder）**：
    *   基于消息传递的 GNN，层间使用 Transformer 风格残差更新。
    *   值节点聚合来自行节点和列节点的消息，行/列节点聚合对应值节点的消息，并经过类型特定的前馈网络。
    *   输出上下文化的节点表示，不做全局池化。
*   **隐式桥接（Latent Bridge / Query-Conditioned Resampler）**：
    *   使用固定数量的可学习查询向量（按行/列/值分组），与编码后节点嵌入及问题嵌入（来自 RoBERTa）进行交叉注意力，生成压缩后的结构软标记 \(Z\)。
    *   通过线性投影匹配 LLM 隐藏维度，注入为 LLM 的前缀嵌入，与文本序列化表示 \(S_{table}\) 和问题共同作为输入。
*   **训练目标**：仅优化图编码器+桥接（约 91M 参数），LLM 参数 \(\Theta_{LLM}\) 冻结；最小化自回归负对数似然。

## 主要结论与发现
1.  **显著性能提升**：GRAB 在所有基准上均优于纯序列化基线，在结构复杂场景（如多表连接、复杂布局）增益最大。
2.  **参数效率与能力保留**：冻结 LLM 的 GRAB 仅 91M 可训练参数，性能接近或超过 LoRA 微调，且避免灾难性遗忘。
3.  **优于静态超图**：查询条件化的潜在桥接在单表设置中全面优于 TAMO，说明动态分配容量更有效。
4.  **小模型超预期**：4B 参数量级流水线可媲美或超越 GPT-5.4-mini（约 100 倍参数），证明显式关系编码可弥补参数规模差距。
5.  **结构化定位 vs. 算术瓶颈**：图编码极大提升行定位和计数能力，但对精确数值运算（如平均值）帮助有限，算术仍依赖 LLM 自身。

## 优点
*   **轻量且高效**：仅训练外部模块，冻结 LLM，可在单 GPU 上完成。
*   **显式捕捉关系**：通过图构造直接建模外键连接、重复值、行列关联，弥补序列化缺陷。
*   **查询条件化的结构读取**：使用问题动态选择重要结构信息，提升压缩效率。
*   **实验设计全面**：多数据集、细粒度压力测试、充分消融与鲁棒性分析，结论扎实。
*   **良好拓展性**：任务无关的结构表示可迁移至事实验证、Text-to-SQL。

## 不足与局限
*   **上下文窗口限制**：仍依赖展平文本，无法处理超过 LLM 上下文长度的超大表格。
*   **数值算术受限**：图将数值桶化，无法进行精确的聚合运算（如平均），成为性能天花板。
*   **模式依赖**：缺失、噪声或歧义的元数据会削弱图构造效果。
*   **前置检索需求**：当前假设相关表已筛选好，未包含表格检索与推理的联合学习。
*   **额外开销**：引入图构造和编码的计算成本，在超大规模数据库上可能成瓶颈。
*   **实验覆盖**：未考虑不完全/脏数据模式下的鲁棒性，未与符号执行引擎深度结合。
